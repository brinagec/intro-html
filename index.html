GCP Commands for Cloud Shell

GCP Command Command Meaning
gcloud compute zone list | grep us-central1 used to display the list of zones close to the zone us-central1
gcloud config set compute/zone us-central1-c configure and set the default zone to us-central1-c
gcloud compute instances create "my-vm-2" / create a virtual machine called my-vm-2 and will be connected to the default subnet
exit close the cloud shell /console
ping my-vm-1 enter into my-vm-1
ssh my-vm-1 log in to my-vm-1
sudo apt-get install nginx-light -y installs a simple web server on my-vm-1 since it is the one you are logged in
sudo nano /var/www/html/index.nginx-debian.html edit the index.html document using the nano editor | you can use the editor of your choice ie vim or code
clear unclutter the console | clean the console
curl http://localhost/ view the code of the web page in the console
exit logout from my-vm-1
curl http://my-vm-1/ check if you are able to view the code of your website from my-vm-2 machine
exit close the cloud shell /console

Follow these Commands for Docker Practice

Command Meaning
yum install docker -y use to install docker on a linux machine and yum is for centos machine
image Static files for an operating machine
instance an actvie operating machine or running image
docker pull ubuntu The command telling docker to get an image of ubuntu from docker library
docker run -it --name pghlibrary ubuntu Telling docker to run the ubuntu image in the interactive terminal of the container using the name pghlibrary. This is the simplest command for creating a container, that is an environment for creating applications, services and files
ls You can list the files found in the instance of ubuntu you are running ontop of docker
touch soup.docx myresearch.docx Using the touch command inside ubuntu instance to create two files soup.docx and myresearch.docx
apt-get update -y It will update the repository for the ubuntu instance pghlibrary
docker ps -a To check the container details, memory, ip address
A developer has asked you to create a development environment for developing an application- WP, DB and php Admin
docker-compose --version To check version of docker compose, which is a tool for creating multiple instances of conterners on docker
vim docker-compose.yml Create and edit the yml file mydb: image: mariadb environment: MYSQL_ROOT_PASSWORD: babatea mywordpress: image: wordpress ports: - 8989:80 links: - mydb: mysql phpadmin: image: corbinu/docker-phpadmin ports: - 8181: 80 links: - mydb: mysql environment: MYSQL_USERNAME: root MYSQL_ROOT_PASSWORD: babatea
docker-compose up -d To check the docker-compose.yml file and start the commands inside this file(run). The environment will be created simply this way in the detached mode, that is, all the three images will be pulled, mariadb, wordpress and corbinu/docker-phpadmin
docker images To check all the docker images(containers) in the the ship
docker ps -a To check details of the images in the docker
docker-compose down To stop and delete the container use down, or else use docker-compose stop , just to stp them
docker rm -f pghlibrary To delete the container not created by docker-compose. To clear memory use docker rmi or docker system prune --all -y , this will clean up the entire images
